Release Notes for OpenCms 6.2 beta

February 9, 2006: This is the public binary BETA release of OpenCms 6.2. OpenCms 6.2 is a major upgrade, and it includes several new features and improvements over OpenCms 6.0.x. We are confident that this beta release is pretty stable, since 6.2 is already used successfully in quite a lot of "live" installations.

Who should use this 6.2 beta version?

This is a beta version. Please don't expect it to be perfectly stable.

Having said that, we believe that this 6.2 beta version is certainly stable enough to be used in a production system. It's likely to be more stable than the current "stable" 6.0.4 release.

If and how you should use this version depends on what your goals are:

  • If you starting a new OpenCms project, you should use this 6.2 beta and not 6.0.4.
  • If you are running a website based on OpenCms 6.0.4, and you want to upgrade, you should wait until the final 6.2 release is available.

When will the "final" release of OpenCms 6.2 be available?

End of Q1 2006.

You can help speeding up the process of development by providing feedback for this release and the following releases, especially regarding bugs. In case you find any bug, please use the OpenCms bugzilla to document it, or at least post a note to the opencms-dev mailing list. The more feedback and code contributions are made, the faster we can fix the issues and finish the development.

The most important new features in this beta release compared to the 6.0.4 version:

Support of new database versions

  • MySQL 5.0.x
  • Oracle 10g

Administration improvements and new features

  • Enhanced Search tool interface for easier search index management
  • New Content check tool to validate file contents and properties
  • New project file list
  • New option to export webuser data
  • New option to export a folder recursively or not
  • All list displays in the Administration backoffice offer a print version
  • Project resources can now be modified in the project management "Edit project" dialog

Additional improvements and new features

  • New file history dialog interface that is able to compare and make a 'diff' of xml pages and xml content including content, properties and file attributes
  • Added new ImageLoader with image processing and scaling capabilities
  • New frontend module to display photo albums conveniently on a website
  • Multi resource selection in explorer view for simultaneous VFS operations on the selected resources, e.g. for copying or moving resources
  • Added <cms:decorate> tag for parsing and decorating HTML content
  • In the Explorer view, it is now possible to select resources by right-clicking on the resource name
  • The Explorer tree view now shows the resource state for new and changed resources
  • It is now possible to select 'individual' or 'shared' property mappings for XML content
  • Workplace explorer resources to display can now be configured using a Collector
  • Added "before publish" event

Resolved Issues

  • Fixed issue #514 & #937: new 'Database Connection Validation' dialog added to setup wizard
  • Fixed issue #1006: it  is now possible to use 3 letter locales (e.g. like "ast")
  • Fixed issue #1020: undo changes now also works with shared properties
  • Fixed issue #1021: file parameter of the cms:property and cms:decorate tag is always toLowerCase()
  • Fixed issue #1024: parent group is shown correctly in the group overview
  • Fixed issue #1027: new folder permissions test added to setup wizard
  • Fixed issue with removing temporary flag after editing content
  • Fixed issue with linebreaks in HTML-Converter
  • Fixed issue in locale handler if non-existant resource has been requested
  • Fixed issue in FlexController that sometimes caused NPE errors during system startup

TODO list to be finished before OpenCms 6.2 is released:

  • Testing and bug fixing - your feedback is required!

Please note: The development of JMX for OpenCms and the Content Relationship Engine (CRE), both features initially intended for the 6.2 release, will not make it in the 6.2 release. We hope to add these features in a later version.

Backward compatibility

OpenCms 6.2 should be highly backward compatible with 6.0.x versions. Certain bug fixes required minor API changes so some adjustments may have to be made for code that uses more advanced API functions such as custom resource types. All Templates should be backward compatible without changes.

To upgrade an existing OpenCms 6.0.x to the new 6.2 release, we will provide an upgrade wizard just for the final 6.2 release. We will not provide a wizard for this OpenCms 6.2 beta release. To upgrade, you should back up your modules and contents and do a fresh installation of the 6.2 beta release.

In case you are using one of the previous development versions (OpenCms 6 alpha, beta or release candidates), there have been a lot of issues fixed so you should definitely update. If you still use OpenCms 5.0.x, you should also update since the final 6.2 version will be much more advanced and stable.

Compatibility with Java versions, Servlet containers and Databases

OpenCms 6.2 beta requires Java 1.4.x or Java 1.5.x. It won't work with Java 1.3.x. We have tested only with SUNs version of the JDK. However, OpenCms has no dependencies to sun.* packages so it should in theory run with all compliant JVMs.

We have tested this release on the Tomcat servlet engine and to some extend on JBoss 4.0. OpenCms 6.2 beta works "out of the box" with Tomcat 4.1.x, Tomcat 5.0.x and Tomcat 5.5.x. Others have reported deploying 6.0.x versions successfully on other servlet containers like BEA Weblogic and SunONE.

On the database side, we have tested with MySQL 4.0.x, MySQL 4.1.x, MySQL 5.0.x, Oracle 9.x and Oracle 10.x. PostgresSQL support has been contributed thanks to Antonio Core. maxDB support has been contributed thanks to Clovis Wichoski and Fabiano Rech. MS SQL server support  has been contributed thanks to Andras Balogh. We still are looking for volunteers to support testing the stability of the PostgreSQL, maxDB and MS SQL server releases.

In case you run OpenCms 6.2 beta on components different from our default components (SUN JDK 1.4.x. / Tomcat 4.1.x, 5.0.x or 5.5.x / MySQL 4.0.x, 4.1.x or 5.0.x / Oracle 9.x or Oracle 10.x), please let us know by posting a message to the OpenCms mailing list. We intend to collect a list of "supported platforms". Also, if you had to patch some lines of code to make OpenCms 6.2 beta run on your platforms, please contribute these so they can be included in the official release.

Regarding version numbers and CVS build tags

In case you want to check out OpenCms from the CVS, please keep the following in mind: The CVS tag for the 6.2 release is "build_6_1_8". Development of further bug fix releases is done in CVS "dev_6_0_1" branch.

You can check out the current development version by selecting the "dev_6_0_1" branch of the CVS. Until we finally publish OpenCms 6.2, we will continue to use 6.1.x as internal CVS build tag numeration.
