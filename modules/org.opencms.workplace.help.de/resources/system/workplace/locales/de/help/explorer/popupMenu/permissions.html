In diesem Dialog können Sie Gruppen oder Benutzern  Berechtigungen für die ausgewählte Ressource zuteilen, Ihre eigenen Berechtigungen für die ausgewählte Ressource und die geerbten Berechtigungen der ausgewählten Ressource für Gruppen und Benutzer einsehen. Demzufolge ist der Dialog in drei Gruppierungen aufgeteilt.

Benuzterberechtigungen


Ansicht der eigenen Berechtigungen.

Hier sehen sie Ihre eigenen Berechtigungen.

Bitte beachten Sie: Sie können in diesem Fenster keine Änderungen Ihrer Berechtigungen an der Ressource vornehmen. Die ist dadurch begründbar, dass das System bei Änderungen nicht wüsste, ob es die neuen Berechtigungen auf die Ressource mit einer Gruppe, in der sich ihr Benutzer befinden oder mit Ihrem Benutzer direkt verbunden werden sollen. Sie können aber in der untersten Gruppierung "Ressourcen Berechtigung" die Gruppe Ihres Benutzers oder Ihren Benutzer hinzufügen und die Art des Zugriffs konfigurieren, um Ihre Zugriffsrechte zu beeinflussen.

Ressourcen Berechtigung

In dieser Gruppierung können Sie die Art des Zugriffs für Benutzer oder Gruppen auf die ausgewählte Ressource konfigurieren. Doch bevor sie die Art des Zugriffs einstellen können, müssen Sie zuerst die Gruppe oder den Benutzer auswählen, für den der Zugriff konfiguriert werden soll.


Auswahl eines Benutzers / einer Gruppe, für die der Zugriff auf die Ressource konfiguriert werden soll.

Sie können die Schaltfläche "Suchen" benutzen, um einen Benutzer oder eine Gruppe auszuwählen, die in Ihrer OpenCms Installation derzeitig vorhanden sind. Nach der Auswähl klicken Sie auf "Hinzufügen". Der Benutzer / die Gruppe erscheint im Dialog und Sie können nun die Art des Zugriffs kontrollieren.

Bitte beachten Sie: Wenn die gewählte Ressource nicht im Pfad Ihres Projektes liegt, können sie keine Berechtigungen manipulieren. Mehr über Projekte und deren Pfade lesen Sie hier.


Konfiguration des Zugriffes einer Gruppe auf die ausgewählte Ressource.

Wenn Sie die Berechtigungen übergeordneter Verzeichnisse für die betreffende Gruppe oder den betroffenen Benutzer unwirksam machen wollen, aktivieren sie das Kästchen "Geerbte überschreiben".

Tip: Mit der Checkbox "Nur intern abrufbar" verhindern Sie, dass die Datei jemals über eine URL "Onlien" aufgerufen werden kann. Auf diese kann dann nur über den OpenCms - Kern zugegriffen werden. Dies ist z.B. sinnvoll, wenn die Datei ein kleiner Teil einer eigentlichen Seite ist, der über eine andere Seite über OpenCms - Funktionalität einbezogen ist. Natürlich will man nicht, dass solche "Schnipsel" aus dem Kontext gerissen und falsch formatiert online einsehbar sind.

Bitte beachten Sie: Vererbte Berechtigungen für eine Gruppe / einen Benutzer auf eine Ressource (siehe unten) sind restriktiv. Wird für eine Gruppe z.B. geerbt, dass Schreiben verboten ist, so ist eine hier konfigurierte Erlaubnis zum Schreiben für die Gruppe hier wirkungslos.

Geerbte Berechtigungen


Anzeige der geerbten Berechtigungen einer Ressource.

Hier können Sie einsehen, welche Berechtigungen für welche Gruppe / welchen Benutzer von der Ressource geerbt wurde. Eine Datei / ein Verzeichnis erbt seine Berechtigungskonfiguration von den darüberliegenden Verzeichnissen. Dies passiert für jede Gruppe / jeden Benutzer separat.
