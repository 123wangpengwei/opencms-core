Wenn Sie mit mehreren Redakteuren zusammenarbeiten, ist es nahezu unmöglich zu erkennen wer wann welche Änderungen an einer Datei oder Verzeichnis vorgenommen hat. Dies ist aber für einen reibungslosen Workflow unerlässlich und nur so ist gewährleistet, dass noch nicht für eine Veröffentlichung vorgesehene Dateien verfrüht publiziert werden.

In OpenCms wird daher der jeweilige Status einer Datei oder Verzeichnisses visuell im Workplace dargestellt.

Farben

  • Blaue Beschriftung – Status „Neu“   
    Dateien und Verzeichnisse werden blau dargestellt, wenn diese neu erstellt oder frisch auf den Server hochgeladen wurden. Diese Dateien und Verzeichnisse sind noch nicht für den Leser im Internet abrufbar.    
  • Rote Beschriftung – Status „Geändert“         
    Dateien und Verzeichnisse werden rot dargestellt, wenn diese geändert oder korrigiert wurden. Der Leser im Internet hat auf die aktualisierten Inhalte noch keinen Zugriff, sondern bekommt die vorherige Version auf dem Monitor angezeigt.
     
  • Schwarze Beschriftung –  Status „Unverändert“     
    Dateien und Verzeichnisse werden schwarz dargestellt, wenn diese im Projekt „Online“ und „Offline“ identisch sind.
 

 

Beachten Sie: Von Ihnen geänderte Dateien sind von Ihnen gesperrt. Mehr zum Thema Sperren von Dateien finden Sie hier.

Beachten Sie: Geänderte Dateien sind nicht "Online" (im Internet) sichtbar und entsprechend gekennzeichnet. Mehr zum Thema Veröffentlichen von Dateien finden Sie hier.
