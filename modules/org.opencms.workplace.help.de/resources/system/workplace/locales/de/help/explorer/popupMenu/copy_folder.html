Etwas komplexer als das Kopieren von Dateien ist das Kopieren von Verzeichnissen. Diese Komplexität ist bedingt durch die Existenz von Verknüpfungen in OpenCms.

Sie haben beim Kopieren von Verzeichnissen drei verschiedene Optionen.


Drei Optionen beim Kopieren von Verzeichnissen.

1.  "Kopiere keine Ressource, erstelle nur Verknüpfungen zu allen Ressourcen im Verzeichnis"

Hierbei wird die Verzeichnisstruktur (alle Unterverzeichnisse) des zu kopierenden Verzeichnisses in ein neues Verzeichnis kopiert. Alle Dateien werden als Verknüpfung kopiert. Eine sinnvolle Aktion, wenn sie eine ganze Site in einer anderen Sprache anbieten wollen.

2.  "Kopiere Resourcen die nicht verknüpft sind, erstelle neue Verknüpfungen für verknüpfte Resourcen"

Hierbei wird die Verzeichnisstruktur (alle Unterverzeichnisse) des zu kopierenden Verzeichnisses in ein neues Verzeichnis kopiert.  Alle enthaltenen Dateien die keine weiteren Verknüpfungen (Geschwister) haben werden inhaltlich kopiert. Für alle Dateien, die mehrere Verknüpfungen haben werden in das neue Verzeichnis kopiert.

Mit dieser Option können Sie verhindern, dass sie Inhalte von Verknüpfungen (mehrfach referenzierte Inhalte) abkoppeln. Vielleicht haben Sie in einem Verzeichnisbaum einige Dateien deren Inhalt einzigartig bleiben muss und bei Änderung überall geändert werden muss (Eine Lizensdatei z. B.). Durch dieses Kopieren bleiben diese Verknüpfungen bestehen, die übrigen Dateien können Sie danach unabhängig von den Orginalen inhaltlich verändern.

3.  "Kopiere alle Resourcen, auch verknüpfte Resourcen"

Hierbei wird die Verzeichnisstruktur (alle Unterverzeichnisse) des zu kopierenden Verzeichnisses in ein neues Verzeichnis kopiert. Nicht nur die individuellen Dateien werden inhaltlich kopiert: Die Vernküpfungen ebenfalls. Dies bedeutet bei Verknüpfungen, dass zwar im neuen Verzeichnis auch Verknüpfungen erscheinen werden, die analog zu den orginalen Verknüpfungen auf den gleichen Inhalt zeigen. Jedoch ist dieser Inhalt nur der kopierte Inhalt der orginalen Verknüpfungen. Wenn sie im kopierten Verzeichnis Änderungen an den Inhalten dieser kopierten Verknüpfungen machen, so bleibt der Inhalt der orginal Verknüpfungen unberührt, lediglich die Inhalte aller Verknüpfungen der Kopie werden manipuliert.
