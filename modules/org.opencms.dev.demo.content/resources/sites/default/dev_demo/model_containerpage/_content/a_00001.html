<![CDATA[How to use the model container page?]]> The model containerpage is a default copy model for the containerpages, where common content which should appear on all new pages is defined. Usually the model containerpage is in folder:

 

{sitefolder}/{sub sitemap path}/_config/models/pages/

 

In order to use the model containerpage, its path has to be set inside of the sitemap configuration. This configuration is applied for this sub sitemap. Please define additionally the title property for the model containerpage, which will appears in the add wizard of the sitemap editor.

 

For this demo example a new model container page is defined in the second tab of the local sitemap configuration:

  • /dev_demo/model_containerpage/_config/models/pages/sitemap_restyp.config
  • /dev_demo/model_containerpage/_config/models/pages/default_with_article.html
  • Title: "Dev Demo Model Containerpage with Article"

You can use the new model containerpage, when new container page is added to the sitemap.

Note: Inside of a sub sitemap only one model containerpage makes sence.
